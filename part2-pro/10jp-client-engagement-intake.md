# 10 - クライアントエンゲージメント受入チェックリスト

新規SAPマイグレーションプロジェクトを受注した際に使用するチェックリストです。アセスメントフェーズ前に、プロジェクトのスコープを確定し移行アプローチを決定するために必要な情報を収集することが目的です。

## 1. クライアントシステム情報

- [ ] 現在のSAPバージョン（ECC 6.0、EHPレベル、SPレベル）
- [ ] 現在のデータベース種別とバージョン（Oracle、MSSQL、DB2 等）
- [ ] 現在のOS（AIX、Linux、Windows）
- [ ] アプリケーションサーバー数 / システムランドスケープ（DEV/QAS/PRD）
- [ ] システム規模 — DBサイズ、ユーザー数、有効な会社コード数
- [ ] インストール済みアドオンまたはISVソリューション
- [ ] Unicodeシステムか否か

## 2. ビジネス背景

- [ ] 目標本番稼働日（ハードデッドラインか柔軟か）
- [ ] 予算規模 / 承認済み予算
- [ ] 社内ITチームのキャパシティ（移行作業をサポートできるか）
- [ ] スコープに影響する事業変化の予定（M&A、組織再編等）
- [ ] 目標展開先：オンプレミス / プライベートクラウド（RISE） / パブリッククラウド
- [ ] 規制・コンプライアンス上の制約（データ所在地、監査要件等）

## 3. スコープ・複雑性の判断材料

- [ ] カスタムABAP（Zプログラム、Zテーブル、エンハンスメント）の量
- [ ] インターフェース・インテグレーション数（RFC、IDoc、API、ミドルウェア等）
- [ ] 使用中の業務プロセス / モジュール（FI、CO、MM、SD、PP、HR等）
- [ ] SAP標準へのカスタム修正の履歴（修正 vs エンハンスメント）
- [ ] サードパーティツールの有無（MES、WMS、BI/BW等）

## 4. 受入後の初期アウトプット

受入完了後、以下を判断できるようにする：
- 移行アプローチ候補の特定（ブラウンフィールド / グリーンフィールド / シェル）→ [12 - アプローチ選定フレームワーク](12jp-approach-selection-framework.md)を参照
- レディネスチェック実施の工数見積もり
- 次のステップ提案：レディネスチェック、ランドスケープワークショップ、フィットギャップ分析

## 5. 確立すべきキーコンタクト

- [ ] クライアントのプロジェクトスポンサー
- [ ] クライアントのITリード / Basis担当者
- [ ] モジュール別ビジネスプロセスオーナー
- [ ] SAPアカウントエグゼクティブ（RISE with SAPが対象の場合）
