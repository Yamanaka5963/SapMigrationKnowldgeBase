# 19 - ハイパーケア & プロジェクト引き渡し

## ハイパーケア概要

ハイパーケアとは、本番稼働直後の集中的なサポート期間です。プロジェクトチームはオンサイトまたはオンコールで問題を迅速に解決し、クライアントの運用チームへ引き渡す前にシステムを安定させます。

標準的なハイパーケア期間：**2〜4週間**（プロジェクト開始時にクライアントと合意すること — 複雑またはスコープが大きいプロジェクトは長め）

## ハイパーケアの準備

本番稼働前に以下を定義しておくこと：

- [ ] ハイパーケアの期間と終了日のクライアント合意
- [ ] サポートチームの名簿とシフトスケジュール（オンサイト vs リモート）
- [ ] 課題の重要度分類とレベル別SLA目標
- [ ] エスカレーションパス（コンサルタント → リード → PM → SAPサポート）
- [ ] 課題管理ツール（Jira、ServiceNow、Solution Manager等）
- [ ] 初週のデイリースタンドアップまたはウォールームのスケジュール

## 課題重要度分類（例）

| 重要度 | 定義 | 対応目標時間 |
|---|---|---|
| **P1 — クリティカル** | システムダウンまたはコア業務プロセスが完全にブロック | 即時（1時間以内） |
| **P2 — 高** | 主要プロセスが影響を受けているがワークアラウンドあり | 4時間以内 |
| **P3 — 中** | 軽微なプロセス問題またはUI上の不具合 | 翌営業日 |
| **P4 — 低** | 機能改善要望または緊急でない質問 | バックログ / 次期リリース |

## 稼働初日の作業

- [ ] システムパフォーマンスとバッチジョブの監視
- [ ] 最初の業務トランザクションにおけるユーザーサポート
- [ ] 全インターフェースの稼働確認（IDoc、バッチジョブ、ミドルウェア）
- [ ] 夜間バッチジョブの失敗ポスティングやエラーログの確認
- [ ] 重要業務プロセスのスモークテスト（受注〜入金、発注〜支払等）

## 引き渡しチェックリスト

プロジェクトをクローズしてクライアント運用チームへ引き渡す前に確認：

### ドキュメント
- [ ] システム設定ドキュメントの納品（トランスポートログ、設定判断記録）
- [ ] カスタム開発ドキュメントの納品（機能仕様書、技術仕様書）
- [ ] 運用手順書の納品（バッチジョブスケジュール、監視ガイド、主要トランザクション）
- [ ] インターフェースドキュメントの納品

### ナレッジトランスファー
- [ ] Basis/ITチームへのシステム管理者トレーニング完了（トランスポート、監視、パッチ適用）
- [ ] キーユーザートレーニングの完了・承認
- [ ] L1サポート向けスーパーユーザーの特定とトレーニング完了
- [ ] クライアントチームが自立してユーザーと権限変更を管理できること

### 未処理項目
- [ ] P1/P2の課題をすべて解決済み、またはワークアラウンドでクライアント承認済み
- [ ] P3/P4のバックログをレビューしてクライアントへ引き継ぎ
- [ ] クライアントチーム向けのSAPサポート契約が有効で、S-Userが割り当て済み
- [ ] SAP Early Watch Alert のスケジュール設定済み（システム健全性モニタリング）

### 正式承認
- [ ] クライアントのプロジェクトスポンサーによるプロジェクト受入書類への署名
- [ ] ふりかえり（レッスンラーンド）セッションの完了
- [ ] 最終プロジェクトレポートの納品

## ハイパーケア後：運用移行

ハイパーケア終了後は、クライアントの社内ITまたはAMS（アプリケーション管理サービス）パートナーが引き継ぎます。確認事項：
- クライアントが本番の全トランスポートルートにアクセスできること
- クライアントがSAP OSSメッセージの起票方法を把握していること
- SAP Solution ManagerまたはSAP Cloud ALMで監視とアラートが設定済みであること
