# 04 - SAP Activate 方法論概要

SAP Activate は、S/4HANA 移行・導入プロジェクトの標準プロジェクト方法論です。レガシーの ASAP 方法論に代わり、アジャイルアプローチを採用しています。

## 6つのフェーズ

| フェーズ | 主な活動 |
|---|---|
| **1. Discover（発見）** | レディネスチェックの実施、システムランドスケープの評価、移行アプローチの特定、初期スコープ確定 |
| **2. Prepare（準備）** | プロジェクトガバナンスの設定、システムインフラ構築、チームオンボーディング、初期バックログ作成 |
| **3. Explore（検討）** | フィット・トゥ・スタンダードワークショップ、ベースラインシステムの設定、ギャップおよびカスタム要件の特定 |
| **4. Realize（実現）** | 設定構築、カスタムコード開発、反復的テスト（単体テスト・統合テスト） |
| **5. Deploy（展開）** | カットオーバー計画、データ移行、ユーザートレーニング、本番稼働 |
| **6. Run（運用）** | ハイパーケア、課題解決、運用チームへの引き渡し |

> 各フェーズの詳細な実行ガイダンスについては、[パート2 — 10: クライアントエンゲージメント受入](../part2-pro/10-client-engagement-intake.md)（英語）以降を参照してください。
