# SAP Migration KnowledgeBase — Project Context

## Purpose
A reference KnowledgeBase for **KEYSTONE** SAP engineer team members who receive and execute SAP migration projects from clients (on-premise SAP ECC → SAP S/4HANA).

## Audience
- **Part 1 (Introduction):** New KEYSTONE members onboarding to SAP migration projects
- **Part 2 (Professional Reference):** Experienced SAP engineers — practical, execution-focused content. Assume deep SAP domain knowledge.

## Structure
```
part1-intro/     ← Introduction for new members (4 files)
```

## Content Guidelines
- Always include references with URLs to original sources
- Distinguish between official SAP sources and community posts
- For official PDFs found via SAP Help Portal, note the parent page in the reference
- Keep Part 1 concise — enough to onboard, not exhaustive
- Part 2 should be practical: checklists, decision frameworks, step-by-step guides

## Key Facts (as of April 2026)
- SAP ECC mainstream maintenance ends **December 31, 2027**
- SAP's strategic direction is **cloud-first** — RISE with SAP (Private Cloud) is the primary recommended path
- S/4HANA On-Premise supported through 2040 but receives no new features
- Three migration approaches: Brownfield (System Conversion), Greenfield (New Implementation), Shell Conversion (Selective Data Transition)
- Current SAP project methodology: **SAP Activate** (6 phases, replaced legacy ASAP)
